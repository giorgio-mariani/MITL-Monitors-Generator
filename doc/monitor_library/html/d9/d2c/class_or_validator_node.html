◆ getValues()

const Signal& OrValidatorNode::getValues ( void  ) const
inlinevirtual

This methods returns the instants between the last two calls at update (offset by a certain value) where the implicit formula is valid. More precisely:

  • Let \(t_i\) be the value of t in the last invocation of update on this.
  • Let \(t_s\) be the value of ts the value of the parameter ts during the invocation of start on this.
  • Let \(t_{i-1}\) be the value of t in the last call of update or if no such call exists \(t_s\)).
  • Let \(\delta\) be the value returned by the invocation of this.minTime().

then this.getValues() returns the instants in (if \(t_s\ge \delta\)) \([t_{i-1} -\delta, t_i - \delta))\) where the formula is true. More generally speaking the instants returned are in the interval:

\[ \big[\max(ts, t_{i-1} - \delta),\max(ts, t_i - \delta)\big) \]

Returns
the instants in \(\big[\max(ts, t_{i-1} - \delta),\max(ts, t_i - \delta)\big)\) where the formula is valid. (the instants are returned as a Signal)

◆ minTime()

RealType OrValidatorNode::minTime ( void  ) const
inlinevirtual

This method return the minimum time that we need to "wait" in order to evaluate a certain instant. More precisely if we want to evaluate the implicit formula for a certain instant \(t\) then we must have the value of the predicates in the formula for at least the instants \([t,t+\delta)\), with \(\delta\) the value returned by this.minTime().

Another reason of why this value is interesting is the following: Let be \(t_i\) the value of the parameter t in the i-th of update then, if \(t_i\) is greater than the value returned by this method, the Signal returned by compute doesn't hav an empty domain (i.e. last > first).

Returns
the minimum time in order to be sure to be able to evaluate a certain instant.

Method that update the instants where the formula is true.

Warning
start must have been already invoked on this and all of its descendants.
Parameters
tinstant (the parameter must have a value greater than the one it had at the last call)
predsvalue of the formula predicate for t
Exceptions
invalid_argumentif preds size is not big enough to contain the values of all the predicates in the formula (i.e. if it exists a descendant of this of type PredicateValidatorNode p with index >= preds.size)

This method update the state of this and all of its descendants so that the next call at compute will return the instants when the formula is true in the interval defined as follows:

  • Let \(t_i\) be the value of t in the current call.
  • Let \(t_s\) be the value of ts the value of the parameter ts during the invocation of start on this.
  • Let \(t_{i-1}\) be the value of t in the last call or if no such call exists \(t_s\)).
  • Let \(\delta\) be the value returned by the invocation of this.minTime().

then the interval for which the formula values are computed is (if \(t_s\ge \delta\)) \([t_{i-1} -\delta, t_i - \delta))\).

More generally speaking the instants for which the formula is evaluated are:

\[ \big[\max(ts, t_{i-1} - \delta),\max(ts, t_i - \delta)\big) \]
